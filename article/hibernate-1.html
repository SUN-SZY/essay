㈠❸HIbernate 多对多关系

Created At：2021-11-18 17:33 Updated At：2022-01-23 15:23

一、映射配置

  • 以角色和用户为例
    • 创建实体类 用户和角色
    • 让两个实体类之间互相表示
      • 表示一个用户可以有多个角色（实体类）
        private Set<Role> setRole = new HashSet<Role>();
      • 表示一个角色有多个用户
        private Set<User> setUser = new HashSet<User>();
    • 基本映射关系
      • 配置角色和用户的类和表对应
      • 配置多对多关系
        • 在用户里面表示所有的角色，使用set标签
        在用户里面表示所有角色，使用set标签
 name属性：角色set集合名称
 table属性：第三张表的名称
 key标签里面配置当前映射文件在第三章表外键名称
 class角色实体类全路径
 columm角色在第三章表的外键名称
 标签为<many-to-many class="" column=""></many-to-many>
    • 配置完毕后在核心映射文件中引入

二、多对多级联保存

  • 添加两个用户，为每个用户添加连个角色
    • 创建角色
    • 实例化对象赋值
    Role r1 = new Role();
r1.setRole_name(///);
r1.setRole_Password(///);
    • 建立关系，把角色放到用户里面
    user---r1/r2
User1.getSetRole().add(r1);
User1.getSetRole().add(r2);
    • 保存用户

三、多对多级联删除

  • 在set标签进行配置 casecade值增添delete
  • 删除用户
    • casecade = “save-update,delete”
       User user = session.get(User.class,1);
 Session.delete(user);

四、维护第三张表关系

  • 用户和角色多对多关系，维护关系通过第三张表维护
  • 让某个用户有某个角色
    • 根据id查询用户和角色
     让某个用户有某个角色
 User kucy = session.get(User.class,1);
 Role role = session.get(Role.class,1);
    • 把角色放到用户里面
    kucky.getSetRole().add(role);
  • 让某个用户没有某个角色
    • 删除角色
    user.getSetRole().remove(role);
